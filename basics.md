Unit test: 
    test logic in classes by programmers to show code level correctness. They should be fast and not dependend on other parts of the system that you don't intend to test

Functional Acceptance Test:
    test use case scenarios on a limited(special created) data set to show that every specified scenario works as specified

User Acceptance Test:
    test use case scenarios on production like data done by representatives of the user to make them formally accept the application

Integration Test:
    test communication paths between different parts of the module
